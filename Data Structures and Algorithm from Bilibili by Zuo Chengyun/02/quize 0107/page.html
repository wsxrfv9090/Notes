Finding the mid is important that you don't use this:
          (L + R)/2

Because if the array is big enough,
L + R could cause stack over flow
and so we should write it like this:
L + (R - L)/2
Then modify it like this:
L + (R - L) >> 1
They are the same thing but it's faster


This function breaks the range from for example: [0,5]
to two ranges like this: [0,2] and [3,5]
then breaks like this: [0,1] & [2] and [3,4] & [5]
